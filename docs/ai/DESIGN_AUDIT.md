# DESIGN_AUDIT — daangn_ad_reporter

> 현재 UI 문제와 향후 점검 기준. 한 화면씩 점검한다. taste-skill의 AI Tells 점검과 병행.

## 점검 기준 체크리스트 (화면마다)
- [ ] 현재 할 일/진행 상태(위자드 단계, 보고서 기간)가 보이는가
- [ ] 로딩 skeleton / 빈 상태 / 오류 상태가 처리됐는가(AI 생성 대기·실패 포함)
- [ ] 주요 행동(생성/내보내기/다음 단계)이 명확한가
- [ ] KPI·퍼널·소재 판단의 색상 코딩이 데이터에 정직한가(과장 없음)
- [ ] 반응형/접근성(키보드·대비·라벨) OK인가
- [ ] taste-skill의 AI Tells(흔한 그라데이션/제네릭 패턴/이모지)가 없는가
- [ ] `dg-` 클래스·Paperlogy·당근 #FF6F0F 적용이 일관된가

## 현재 관찰 (구조 기준 가설)
- 4단계 위자드(`planning_wizard.py`) → 단계 진행 상태/실패복구 표시 점검 우선.
- 성과 보고서(`report.py` + `funnel_widget.py`) → 퍼널·MAX CPA·소재 ON/OFF 시각화 정직성 점검.
- 썸네일 생성 → 대기/실패 상태 처리 점검.
