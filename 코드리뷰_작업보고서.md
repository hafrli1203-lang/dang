## 1. 프로젝트 개요

당근마켓 광고주를 위한 성과 보고서/기획서 자동 생성 도구.
- 3개 페이지: 프로젝트 관리 → 기획서 생성 → 성과 보고서 생성
- AI 엔진: Claude (Anthropic) / Gemini (Google) 선택 또는 동시 비교
- 출력: DOCX 문서 + 차트 (matplotlib)
- 배포: PyInstaller + Inno Setup 인스톨러

## 3. 수정 내역

### 3-1. build.py — import os 위치 오류 (CRITICAL)

**수정 전:**
```python
# line 12
import subprocess
import sys
import shutil
from pathlib import Path

# ... line 48에서 os.pathsep 사용 → NameError

# line 100
if __name__ == "__main__":
    import os  # ← 여기서만 import
```

**수정 후:**
```python
# line 12
import os  # ← 최상단으로 이동
import subprocess
import sys
import shutil
from pathlib import Path

# line 99
if __name__ == "__main__":
    # import os 제거 (중복)
    args = sys.argv[1:]
```

---

### 3-2. app/pages/report.py — tempfile 리소스 누수 (HIGH)

**수정 전:**
```python
def _make_report_docx_bytes(project, rows, kpi, content):
    import tempfile
    from pathlib import Path as _Path
    tmp = _Path(tempfile.mkdtemp())  # ← 생성만 하고 삭제 안 함
    out = tmp / "report.docx"
    build_report_docx(...)
    return out.read_bytes()
```

**수정 후:**
```python
def _make_report_docx_bytes(project, rows, kpi, content):
    import tempfile
    with tempfile.TemporaryDirectory() as tmp_dir:  # ← 자동 정리
        out = Path(tmp_dir) / "report.docx"
        build_report_docx(...)
        return out.read_bytes()
```

변경 포인트:
- `tempfile.mkdtemp()` → `tempfile.TemporaryDirectory()` 컨텍스트 매니저
- `from pathlib import Path as _Path` 제거 (이미 line 4에서 `Path` import됨)

### 3-3. app/updater.py — 로깅 불일치 (MEDIUM)

**수정 전:**
```python
import logging
logger = logging.getLogger(__name__)
# ...
logger.debug("Update check failed: %s", exc)
```

**수정 후:**
```python
from app.logger import get_logger
_log = get_logger("updater")
# ...
_log.debug("Update check failed: %s", exc)
```

효과: updater 로그가 앱의 "최근 로그 보기" UI 패널에 표시됨

### 3-4. app/pages/report.py — 파서 조건 가독성 (LOW)

**수정 전:**
```python
elif "next action" in header or ("액션" in header and "다음" not in header.replace("다음 실험", "")):
```

**수정 후:**
```python
elif "next action" in header or "다음 액션" in header:
```

"다음 실험"은 별도의 `"실험" in header` 분기에서 처리되므로 안전한 단순화.
